# FSA Marketing Website — Project Overview

## Purpose

This repo is the static marketing website for **Full Steam Ahead (FSA)**, a 2nd class Power Engineering exam prep platform for Canadian operators. It lives at `fullsteamahead.ca`.

The repo has four distinct concerns:

- **Marketing website** — pure HTML/CSS, no build step, no templating engine, served via nginx + Docker + Cloudflare
- **SEO content hub** — 20 articles organized in a pillar/cluster architecture across three topic areas
- **AI article generation pipeline** — Python scripts that call Claude Sonnet via OpenRouter to generate, fill, and validate article HTML against a manifest
- **Publishing & SEO tooling** — Cloudflare cache purge, Google Search Console sitemap submission + URL indexing, Docker build/deploy workflow

## AI Article Generation Pipeline

Articles are generated by `scripts/generate_article.py`. The pipeline is manifest-driven: `scripts/articles_manifest.json` is the single source of truth for every article's slug, primary keyword, internal links, related cards, breadcrumb path, intro text, and published date.

### Generation sequence

Each article requires three sequential API calls to OpenRouter (model: `anthropic/claude-sonnet-4-6`):

1. **Research brief** — asks the model to summarize verified, current facts relevant to the article topic. The `SOPEEC_FACTS` constant (updated for January 2025) is injected into every research prompt as verified ground truth, ensuring exam details (paper format, pass mark, time limits) are never hallucinated or hedged incorrectly.

2. **Article body HTML** — uses the research brief plus the manifest entry (keyword, internal links, CTA placement, word target) to generate the full article body. Output is raw HTML intended for insertion inside `<article class="article-body">`. No markdown fences.

3. **Meta description** — generates a 140–160 character meta description containing the primary keyword.

### Template filling

`fill_template()` reads `articles/_template/index.html` and substitutes every `REPLACE:` placeholder with values from the manifest entry and the three API outputs. The result is written to `articles/{slug}/index.html`. A `sanitize_unicode()` pass converts typographic characters (em dashes, smart quotes, ellipses) to safe HTML entities before writing.

### CLI usage

```bash
# Single article by manifest ID
python3 scripts/generate_article.py P1

# All articles in strategy order
python3 scripts/generate_article.py --all

# Pillar articles only
python3 scripts/generate_article.py --pillars

# One cluster
python3 scripts/generate_article.py --cluster A

# Preview all three prompts without calling the API
python3 scripts/generate_article.py P1 --dry-run
```

Requires: `OPENROUTER_API_KEY` in environment.

## SEO & Publishing Workflow

### Sitemap

`sitemap.xml` is manually maintained — no script generates it. It must be updated by hand when a new article is published.

### Google Search Console

`scripts/submit_to_google.py` handles two tasks:

- Submits `https://fullsteamahead.ca/sitemap.xml` to Search Console
- Calls the URL Inspection API to request indexing for all 21 URLs (homepage, articles index, and all 20 article pages)

OAuth credentials live in `scripts/google_client_secret.json`; the access token is cached in `scripts/google_token.json` after the first browser-based consent flow.

### Cloudflare cache purge

`scripts/purge_cloudflare.sh` calls the Cloudflare API to purge the entire zone cache (`purge_everything: true`), forcing CDN nodes to re-fetch fresh content from the origin. Reads `CF_ZONE_ID` and `CF_API_TOKEN` from `.env` or environment; falls back to `~/.env`.

## Deployment

The site runs as a single Docker container (`fsa-website`) on the host, behind a Cloudflare tunnel. nginx serves `index.html` at `/` and article pages via a `try_files $uri/index.html` rule. Port mapping: `8087:80`.

The container joins an external Docker network named `cloudflare`. This network must exist on the host before `docker compose up` will succeed:

```bash
docker network create cloudflare  # once per host
```

### Rebuild and redeploy

```bash
docker compose build && docker compose up -d
./scripts/purge_cloudflare.sh
```

## Typical Operational Workflows

### Publish a new article

```bash
# 1. Generate the article HTML from the manifest
python3 scripts/generate_article.py B3

# 2. Review and edit articles/ai-tutoring-power-engineering-study/index.html

# 3. Update sitemap.xml — add the new article URL by hand

# 4. Rebuild and redeploy
docker compose build && docker compose up -d

# 5. Purge CDN cache
./scripts/purge_cloudflare.sh

# 6. Submit to Google Search Console
python3 scripts/submit_to_google.py
```

### Deploy a site change (no new article)

```bash
# 1. Edit index.html, styles.css, or any static file

# 2. Rebuild image and restart container
docker compose build && docker compose up -d

# 3. Purge CDN cache
./scripts/purge_cloudflare.sh
```
